Firewalls

Firewall

Packet-filtering Firewalls
· Most common kind of firewalls · Linux: iptables and nftables. Windows: Microsoft Defender Firewall · Usually active by default
· Behave based on list of rules matching packets · IF packet matches THEN action · Action: ACCEPT, REJECT, DROP

Advantages and Disadvantages of Packet-filtering Firewalls
+ Powerful protection for entire network: filtered packets never reach devices
+ Significantly reduce attack surface + Simple rule matching  very efficient + Widely available, many resources: documentation, GUIs, . . .
- Usability: firewall cannot block everything - Possible to hide traffic as different protocol/port
- Bugs in rules - Need to balance protection and rule complexity
- e.g., whitelisting every single authorized IP - Only works if traffic transits through firewall
- e.g., using mobile data instead of WiFi
+ Overall, widely deployed, especially for large networks

Proxy Firewall
· Application-level (OSI layer 7)
· (Forced) intermediary between A and B: 1. Proxy receives request from A 2. Proxy makes similar request to B 3. Proxy receives response from B 4. Proxy crafts and send response to A
· Can be explicit or transparent · Transparent: one of the devices does not know proxy is in place
· Can require authentication

Advantages and Disadvantages of Proxy Firewalls
· Application knowledge + Logging  accountability + Caching  bandwidth reduction + More intelligent filtering - Application-specific
+ Aggregation of clients under same endpoint  anonymity - Complex: can degrade performances - Usually requires specific setup for one end of connection
+ Allows controlled connections to a service without service needing to implement authentication (e.g., SOCKS proxy) - Need to trust proxy with handling sensitive data
- Basically, an authorized man-in-the-middle attack

Tiered Networks

Intrusion Detection Systems

IDS vs. I(D)PS
· Intrusion detection is the process of monitoring and analyzing system events to identify and report attempts to access resources in an unauthorized manner.
· Intrusion prevention systems additionally take action in response to detected (unauthorized) activity to stop or contain the activity.
· Actions include: · Alter packets (e.g., strip malware) · Sandbox users/connections · Shut down connections, processes, systems · Redirect to honeypots

Event Sources for Intrusion Detection
· NIDS: Network-based IDS · Events derived from listeners at strategic points · Router, gateways, switches · Use packet sniffing to observe in-transit packets · Snort, Suricata
· HIDS: Host-based IDS · Events derived from kernel operations, application logs, filesystem changes, · system calls, host network accesses . . . · OSSEC

Signature-Based Event Analysis
· Define signatures, i.e., malicious patterns · Condition lists, e.g., fields in packet headers · Raw byte sequences
· Match events against signatures
· alert icmp $EXTERNAL_NET any -> $HOME NET any (dsize: 0; itype: · 8; msg:"ICMP PING NMAP";) · nmap scans a network by sending empty ICMP "ping" packets and
seeing · which hosts responds  obtains network topology · Rule detects such packets and can warn about initial step of an attack

Advantages and Disadvantages of Signature-Based IDS
+ Fast + Accurate + Signatures can be shared: learn from others - Signatures need to be updated by somebody - Malicious actors try to obfuscate their signatures

Anomaly-Based Event Analysis
· Build profiles of legitimate users · Identify deviations from these profiles
· Examples: · Number of (failed) login attempts · Number of accesses/deleted files (within a time window) · Commands executed · Resources consumed (e.g., CPUs)

Advantages and Disadvantages of Anomaly-Based IDS
+ Detects "never seen before" attacks
+ "Automatic" learning
- Difficult to configure - Selecting features - Ensure intruder-free training - Usual problems of training a ML/anomaly detection model  tradeoffs
- Legitimate users can behave abnormally  false positives

Base Rate Fallacy
· We can adjust the sensitivity of IDSs (usually) both during training and during operation · Can be made more sensitive  better detection but false positives · Can be made less sensitive  might miss some malicious activity
