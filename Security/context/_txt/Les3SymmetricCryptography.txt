Important goals of cryptography
À Confidentiality = keeping information secret for everyone except for authorized entities
À Authentication À of entities = assuring the source of information À of data = assuring that the information has not been altered by unauthorized entities
À Non-repudiation = assuring that actions or commitments in the past cannot be denied

Classification of cryptography
À Symmetric-key cryptography: À Mainly for confidentiality À Sender and receiver have the same secret key À Block ciphers and stream ciphers
À Public-key cryptography: À Mainly for authentication (digital signatures) and key agreement À Sender and receiver have a different key (= public and private key)
À Hash functions: À Mainly for the authentication of data À With or without secret key

One-time pad
À "The perfect cipher" is the only proven method for cryptography that makes an unbreakable encryption possible. Other names are the onetime pad or the Vernam scheme (Gilbert Vernam, in 1917).
À The key is completely random and has the same size as the plaintext. This makes the one-time pad impractical.
À Example:
01011 XOR 10111 = 11100 (plaintext XOR key = ciphertext) 11100 XOR 10111 = 01011 (ciphertext XOR key = plaintext)

Properties of hash functions
The ideal hash function for cryptographic applications has the following properties:
À It is easy to calculate the hash of a message
À Given the hash value h, it is difficult to find a message that has h as a hash value = pre-image resistance
À Given the message m1, it is difficult to find a message m2 that has the same hash value as the hash value of the message m1 = second pre-image resistance
À It is difficult to find two different messages with the same hash value = collision resistance

The use of a secret key
À Hash functions without a secret key are used to assure data integrity. They allow to detect changes in the transmitted messages.
À Hash functions can also be used for authentication (of data). In this case, a secret key is used, together with the message, and we call the function a Message Authentication Code (MAC).

Message Authentication Code ¡ MAC

Examples of MAC algorithms
Commonly used MAC algorithms are: À CMAC: MAC based on a symmetric-key block cipher À HMAC: MAC based on a hash function
